# Полный план запуска проекта «Детская лига» (от MVP до платформы)

## Обзор проекта
Веб и мобильная платформа для управления детскими спортивными клубами. Веб-сайт обслуживает **все роли**: администратор, тренер, родитель, игрок. Мобильное приложение дублирует функции веб для удобства доступа на ходу.

## Текущее состояние проекта
- **База данных:** 100% готова (4 миграции, 40+ таблиц, ENUM, JSONB, индексы)
- **Документация:** 95% готова (C4 L1-L3, ER-диаграммы, journey maps, sequence diagrams, data flow)
- **Модели Eloquent:** ~20% (15 стабов, требуется синхронизация с миграциями + 15+ новых)
- **Бизнес-логика:** ~5% (контроллеры пустые, сервисного слоя нет)
- **API endpoints:** ~1% (маршруты не реализованы)
- **Тесты:** 0%

## Фаза 0: Подготовка инфраструктуры (2 недели)
**Старт:** День 1
**Завершение:** День 15
**Команда:** 1 backend-разработчик + 1 DevOps

### Задачи
1. **Сервер и окружение** (5 дней)
   - Настройка production/staging серверов (Ubuntu + Nginx + PHP 8.2 + PostgreSQL 15)
   - Настройка CI/CD (GitHub Actions / GitLab CI): линтинг, тесты, деплой
   - Настройка S3-совместимого хранилища (MinIO / AWS S3)
   - Настройка Redis для очередей и кэширования

2. **Laravel: структура и безопасность** (5 дней)
   - Установка и настройка Laravel Sanctum (API-токены)
   - Настройка RBAC (роли: admin, coach, parent, player)
   - Middleware для проверки прав доступа
   - CORS-настройка для мобильного приложения

3. **Синхронизация моделей с БД** (3 дня)
   - Исправление всех моделей: имена таблиц, колонок, relationships
   - Создание недостающих моделей (15+ штук)
   - Удаление 127 нерелевантных Livewire-компонентов
   - Создание базового сервисного слоя (FileService, AuthService)

4. **PostgreSQL: миграции** (2 дня)
   - Применение и проверка всех миграций
   - Создание seed-данных для справочников
   - Проверка целостности FK, индексов, ENUM

### Результат
Рабочая инфраструктура, чистая кодовая база, все модели синхронизированы.

## Риски и митигация

| Риск | Вероятность | Влияние | Митигация |
|------|------------|---------|-----------|
| Задержка OAuth-интеграции | Средняя | Средняя | Сначала email-auth, OAuth добавить параллельно |
| Сложность live-матча (WebSocket) | Высокая | Высокая | Начать с polling, перейти на WS позже |
| Ревью App Store > 2 недель | Средняя | Средняя | Подавать заранее, иметь TestFlight/beta |
| Нехватка мобильных разработчиков | Средняя | Высокая | Web-first стратегия снижает зависимость от mobile |
| Несовместимость моделей с БД | Высокая | Критическая | **Уже выявлено** — исправить в Фазе 0 |

## Ключевые метрики готовности MVP Web

- [ ] Все 4 роли могут войти через веб-сайт
- [ ] Администратор может создать клуб, команду, добавить игроков и тренеров
- [ ] Тренер может создать тренировку и отметить посещаемость
- [ ] Родитель может подтвердить/отклонить присутствие ребёнка через веб
- [ ] Игрок может просмотреть расписание и статистику через веб
- [ ] Тренер может провести live-матч с фиксацией событий
- [ ] Родитель получает Push-уведомления о голах и итогах матча
- [ ] Статистика отображается для всех ролей
- [ ] Экспорт в Excel и iCal работает
